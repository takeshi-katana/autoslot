# KagamiAuto

KagamiAuto — веб-система для организации онлайн-записи и управления расписанием автосервиса. Приложение объединяет клиентский сценарий бронирования и рабочее пространство сотрудников, помогая вести каталог услуг, контролировать загрузку и обрабатывать заявки в одном интерфейсе.

Система подходит для автосервиса, которому необходимо сократить количество ручных операций при записи клиентов и снизить риск пересечения заявок. Клиент выбирает услугу, дату и доступное время, указывает контактные данные и сведения об автомобиле. Сотрудник работает с поступившими заявками через защищённую административную панель.

## Возможности

### Для клиентов

- просмотр услуг, длительности и ориентировочной стоимости;
- выбор даты и свободного временного интервала;
- создание записи с контактными данными и номером автомобиля;
- просмотр собственных записей;
- проверка текущего статуса заявки;
- отмена активной записи.

### Для сотрудников

- защищённый вход в административную панель;
- просмотр записей за выбранный период;
- фильтрация заявок по статусу;
- поиск по клиенту, телефону, автомобилю или услуге;
- подтверждение и отмена записей;
- управление каталогом услуг;
- контроль количества заявок по статусам.

## Логика расписания

Доступные интервалы рассчитываются с учётом продолжительности выбранной услуги, рабочего дня и уже существующих активных записей. Заявки со статусами `pending` и `confirmed` блокируют пересекающееся время, а отменённые заявки не влияют на доступность.

Защита от двойного бронирования реализована на двух уровнях:

- предварительная проверка пересечений в бизнес-логике приложения;
- ограничение PostgreSQL, предотвращающее конкурентную запись на пересекающиеся интервалы.

## Статусы записей

| Статус | Назначение |
| --- | --- |
| `pending` | Заявка создана и ожидает обработки |
| `confirmed` | Время подтверждено сотрудником |
| `cancelled` | Заявка отменена и больше не блокирует расписание |

## Основные страницы

| Адрес | Назначение |
| --- | --- |
| `/` | Главная страница системы |
| `/book` | Создание клиентской записи |
| `/my-bookings` | Просмотр и отмена клиентских записей |
| `/cameras` | Просмотр камер сервисных зон |
| `/admin/login` | Вход для сотрудников |
| `/admin/bookings` | Управление расписанием и заявками |
| `/services` | Управление каталогом услуг |
| `/health` | Проверка доступности приложения |

Административная панель и управление услугами доступны только после авторизации сотрудника.

## Структура приложения

Основная бизнес-логика разделена на контексты:

- `Autoslot.Services` — каталог и параметры услуг;
- `Autoslot.Bookings` — создание, обновление и контроль записей;
- `Autoslot.Scheduling` — построение и проверка временных интервалов;
- `Autoslot.Bookings.CustomerLookup` — клиентский поиск и отмена записей.

Пользовательские интерфейсы реализованы на Phoenix LiveView. Данные хранятся в PostgreSQL, а ограничения целостности поддерживаются как changeset-валидациями, так и правилами базы данных.

## Локальный запуск

### Требования

- Elixir и Erlang/OTP;
- PostgreSQL;
- Git.

### Установка

```bash
git clone https://github.com/takeshi985/KagamiAuto.git
cd KagamiAuto
mix setup
```

Для ручной подготовки окружения можно выполнить:

```bash
mix deps.get
mix ecto.create
mix ecto.migrate
mix run priv/repo/seeds.exs
```

### Учётная запись сотрудника

Перед запуском задайте данные для входа в административную панель:

```powershell
$env:AUTOSLOT_ADMIN_USERNAME="admin"
$env:AUTOSLOT_ADMIN_PASSWORD="change-me"
```

В production переменные `AUTOSLOT_ADMIN_USERNAME` и `AUTOSLOT_ADMIN_PASSWORD` обязательны. Используйте уникальный пароль и храните его в настройках окружения платформы развёртывания.

### Запуск сервера

```bash
mix phx.server
```

Приложение будет доступно по адресу `http://localhost:4000`.

## Проверка работоспособности

Полная локальная проверка проекта:

```bash
mix precommit
```

Команда проверяет компиляцию без предупреждений, форматирует исходный код и запускает тесты.

Для проверки запущенного приложения используется endpoint:

```text
GET /health
```

Исправный экземпляр возвращает HTTP `200` и ответ `ok`.

## Конфигурация production

Для запуска необходимы переменные окружения Phoenix и PostgreSQL, включая:

- `DATABASE_URL`;
- `SECRET_KEY_BASE`;
- `PHX_HOST`;
- `AUTOSLOT_ADMIN_USERNAME`;
- `AUTOSLOT_ADMIN_PASSWORD`.

Перед запуском новой версии должны быть применены миграции базы данных. HTTPS принудительно включается production-конфигурацией приложения.

## Контроль качества

GitHub Actions запускает автоматическую проверку изменений. CI компилирует проект, проверяет форматирование и выполняет тесты, включая сценарии бронирования, управление услугами, маршруты LiveView и расчёт расписания.

Система развивается по мере изменения операционных процессов: архитектура позволяет расширять правила расписания, подключать уведомления, интеграции с внешними системами и дополнительные роли сотрудников.
